## 잘 된 것들 (Successes)

### 1. 스토리 분리와 팀 내 충돌 없는 진행
스토리마다 '이번 스토리에서 한다 / 하지 않는다' 경계 표가 명확했고, 실제로 그 경계가 지켜졌다. Story 1.1에서 GitHub Actions를 `.gitkeep`만 두고 1.5로 넘긴 것, Story 1.2에서 `VarcoInterface` Protocol만 정의하고 구현체는 1.4로 미룬 것이 대표 사례. 스토리 간 기술적 의존성이 자연스러운 빌드업 순서(1.1→1.2→1.3→1.4→1.5)로 설계되어 팀 내 충돌이 없었다.

### 2. 코드 리뷰 프로세스 효과
5개 스토리에서 총 21건의 패치가 적용되었다. 주요 사례:
- Story 1.3: Redis/PostgreSQL 호스트 포트를 `127.0.0.1`로 한정 (LAN 노출 차단)
- Story 1.4: `detections.confidence` 범위 검증 누락, VarcoMock 오류 처리 개선, CSV CRLF 인코딩 문제
- Story 1.4: `infra/docker-compose.yml`에 postgres healthcheck 추가

### 3. `shared/` 패키지 설계 품질
표준 라이브러리만으로 `CrawlEvent`, `VarcoInterface`, `RedisConfig`, `structured_logger` 구현. 외부 의존성 없이 crawler/detection 양쪽에서 `from shared.xxx import yyy` 즉시 사용 가능. Epic 2~4에서 계약 충돌 없는 개발 기반 마련.

### 4. 적응적 의사결정과 기록
외부 도구가 계획 버전을 거부했을 때 빠르게 결정하고 Change Log에 이유를 남겼다. (Spring Boot 3.4.x EOL → 3.5.0 전환, Foojay Toolchain Resolver 추가, H2 testRuntimeOnly 추가)

### 5. 상세한 문서화
스토리마다 Dev Notes, 검증 명령, Anti-Patterns, 참조 링크가 체계적으로 작성됨. 다음 스토리에서 'Previous Story Intelligence' 섹션으로 지식이 이어짐.

## 어려웠던 것들 (Challenges)

### 1. 계획 버전과 실제 환경 불일치 (5스토리 중 4번 발생)

| 스토리 | 계획 | 실제 발생 |
|---|---|---|
| 1.1 | Spring Boot 3.4.x | Spring Initializr가 3.4 EOL로 거부 → 3.5.0으로 전환 |
| 1.1 | Java 21 로컬 설치 가정 | Java 25 환경 → Foojay Toolchain Resolver 추가 필요 |
| 1.2 | `setuptools.backends.legacy:build` | Python 3.14 환경에서 미지원 → 3번 시도 끝에 해결 |
| 1.3 | 포트 6379, 5432 기본 | 기존 next-redis/next-postgres 컨테이너 점유 → 충돌 |
| 1.5 | crawler/detection 로컬 lint+pytest | PEP 668 brew Python 차단 → GitHub Actions로만 검증 |

**근본 원인:** 아키텍처 결정 시점과 실제 구현 시점 사이의 시간 차. 개발 환경이 bleeding edge (Python 3.14, Java 25, Node 25)인 반면 공식 지원 버전(3.11, 21, 20.19)과 불일치.

### 2. 외부 검증 불가로 인한 불확실성
Story 1.5에서 crawler/detection CI가 로컬에서 검증되지 못해 GitHub Actions 실제 실행 결과에 의존. "되는 것 같다"는 수준의 확인으로 마무리됨.

## 주요 통찰 (Key Insights)

1. **스토리 분리 품질 = 팀 마찰 없는 진행** — 경계가 명확할수록 충돌이 없다
2. **외부 툴링 버전은 착수 전에 반드시 확인** — 계획 단계의 버전이 구현 시점에 유효하지 않을 수 있다
3. **코드 리뷰가 실제 문제를 잡는다** — healthcheck 누락, confidence 범위, CSV 인코딩 모두 리뷰에서 발견됨
4. **합성 데이터는 구조 검증용** — 실제 Precision 달성을 위해서는 실제 중국어 커뮤니티 게시글 라벨링 필요
